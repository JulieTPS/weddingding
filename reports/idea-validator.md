*Généré le 27/06/2026*

---

## ÉTAPE 0 — DÉTECTION DU TYPE

**Hybride physique + SaaS web.**

Le produit combine deux composantes : un produit physique (100 cartes imprimées avec puce NFC) et une plateforme web (dashboard couple + site de mariage hébergé). Ce n'est pas une app mobile (pas de téléchargement requis — le NFC ouvre un site web natif).

**Point d'entrée recommandé :** Construire d'abord le dashboard web et le moteur de site mariage, puis ajouter la brique physique (impression + NFC) en phase 2. Raison : la partie web est itérable sans délai logistique, et une commande pilote peut valider l'opérationnel avant de scaler.

### Source 4 — Willingness to Pay

- **Mariages France 2024 : 247 000** (INSEE). [Source](https://laplace-groupe.com/publications/chiffre-du-mois/le-chiffre-du-mois-247000-soit-le-nombre-de-mariages-celebre-en-2024-en-france)

- **Budget faire-part :**
  - Standard : 1€–2,50€/unité → 100–250€ pour 100 cartes
  - Semi-personnalisé : 2,50€–5€/unité → 250–500€ pour 100 cartes
  - Sur-mesure illustré : 5€–12€/unité → 500–1 200€ pour 100 cartes
- **Sites de mariage : GRATUITS** (Mariages.net, WithJoy, Zola). Les couples ne paient pas pour un site mariage aujourd'hui.

Sources : [Marin et Toinette](https://marinettoinette.com/prix-faire-part-mariage/), [Mariee.fr](https://www.mariee.fr/budget-mariage/faire-part/)

**Analyse :** Les couples paient €200-600 pour des faire-parts et €0 pour un site web. Le bundle NFC+site à €199-299 doit être vendu comme "faire-part premium amélioré", jamais comme "faire-part + site web".

- Coût petit volume : 0,30€–0,70€/puce
- Coût bulk 1000+ : 0,03€–0,05€/puce

**Stack MVP :** Next.js + Supabase + Cloudflare R2. Features : dashboard couple, site public (RSVP, programme, photos, livre d'or, compte à rebours), routing URL unique par couple, admin commandes.

**Estimation build MVP avec Claude Code : 2 à 4 semaines.**

## ÉTAPE 4 — MARC DESJARDINS (Devil's Advocate)

**Q1 — "La NFC c'est gadget. Les couples vont vraiment payer pour ça ?"**
Risque réel. Vendre sur l'expérience (tap → wow), jamais sur la technologie.

**Q2 — "Tous les invités n'ont pas un iPhone compatible NFC."**
Gérable : iPhone 7+ suffit (2026 = < 5% du parc en dessous). QR code au dos = fallback.

**Q3 — "CottonBird ou Mariages.net peut lancer ça dans 6 mois."**
Risque moyen. Intégrer NFC physique dans un e-commerce est plus complexe qu'il n'y paraît. Window de 12-18 mois existe.

**Q4 — "Budget 150€, ça tient pas pour du physique."**
Partiellement vrai. 150€ = 1 commande pilote test. Stratégie imposée : 0% de stock, 100% paid before production.

**Q5 — "Les couples se marient une fois. Pas de rétention."**
Vrai structurellement. 247 000 mariages/an = marché qui se renouvelle. Extension possible : naissances, baptêmes.

**Q6 — "Personne ne tape un NFC dans la vraie vie."**
En train de changer avec le paiement sans contact et les cartes de visite NFC. L'effet WOW du tap > QR code.

**Q7 — "30-50 commandes max, c'est de l'artisanat."**
Vrai à court terme, délibéré. Phase 1 = validation. Phase 2 = industrialisation.

**Q8 — "Les imprimeurs vont te marginaliser."**
Risque faible. Tu es leur client, pas leur concurrent.

**Q9 — "Si tu fermes dans 2 ans, les couples perdent leur site ?"**
Risque de confiance réel. Mitigation : 2 ans d'hébergement inclus dans l'offre + clause export PDF.

**Q10 — "Le marché est saisonnier."**
~60% des mariages en mai-septembre. Commandes 3-6 mois avant = pic janvier-avril. Mitigation : étendre aux baptêmes, anniversaires.

## SYNTHÈSE FINALE

**Score global : 7.075/10 → CONDITIONNEL**

### Les 3 conditions pour passer en GO

1. **Valider le prix avec de vrais signaux terrain** : poster dans groupes Facebook "Mariage 2026" un mock-up + sondage "paieriez-vous 199€ ?". Si 10 réponses positives en 48h → signal vert.

2. **Lever l'ambiguïté de la proposition de valeur** : tester angle (A) "Faire-parts premium avec NFC" vs angle (B) "Site de mariage + invitations en un seul produit".

3. **Résoudre la saisonnalité dès le départ** en ciblant aussi fiançailles, baptêmes, anniversaires pour les mois creux.

**Risque principal :** Les couples perçoivent les outils digitaux de mariage comme gratuits. Le bundle doit justifier un premium sur le prix du faire-part, sans que le couple ait l'impression de "payer pour un site web".
